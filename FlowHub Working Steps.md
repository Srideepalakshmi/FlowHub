FlowHub Working Steps

Register – A new user creates an account by entering the required details and selecting a role.

Login – The user logs in using the registered credentials.

Role-Based Dashboard Access – After login, the system redirects the user to the dashboard based on the assigned role:

Employee

Manager

Finance

CEO

Admin

Employee Request Submission – The employee submits a new approval request through the workflow form.

Manager Approval Stage – The submitted request moves to the Manager dashboard, where the manager can approve or reject it.

Finance Approval Stage – After manager approval, the request moves to Finance for the next approval.

CEO Final Approval Stage – After finance approval, the request moves to the CEO for final approval or rejection.

Status Update – If approved by the CEO, the request status changes to Completed. If rejected at any stage, the status changes to Rejected.

Admin Monitoring – Admin can monitor all requests, approvals, rejected requests, completed workflows, and failed executions.

Workflow Management – Admin can create workflows, define approval steps, and configure rules.

Execution Tracking – Every workflow action is recorded and tracked step by step.

Settings Control – Admin manages workflow settings, notifications, retries, and execution controls.
